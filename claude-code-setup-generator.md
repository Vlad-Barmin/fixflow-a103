## Философия этого документа

Этот файл — **готовый промпт для Claude AI**. Вы не пишете его сами — копируете и используете.

**Принцип работы:**
- Вы — архитектор и коммуникатор
- Claude — исполнитель: читает промпт, делает работу
- Ваша задача — описать контекст и скопировать правильный промпт

**Workflow:**
```
Вы → копируете промпт → загружаете в Claude AI (web)
       ↓
       прикрепляете свой контекст (SPEC.md)
       ↓
Claude → выполняет задачу по промпту
       ↓
Вы → получаете результат (пакет настроек) → переносите в Claude Code
```

Промпт уже отлажен. Не нужно его улучшать с первого раза — просто используйте.

## ЧАСТЬ 0: ОПРЕДЕЛИ ТИП ПРОЕКТА

Перед генерацией, определи по спецификации — есть ли в проекте продакшн AI-агенты (агенты, которые будут работать для конечных пользователей SaaS-платформы).

Признаки наличия продакшн AI-агентов в спецификации:
- Упоминаются "AI-ассистенты", "чат-боты", "автоматизация через AI"
- Платформа обрабатывает данные пользователей через LLM
- Есть интеграции с внешними сервисами через AI (email-агент, агент поддержки, агент-аналитик)
- Пользователи создают или настраивают собственных агентов

Если да — добавь в генерацию:

## Два подхода к созданию продакшн агентов

### Подход A: Anthropic Agent SDK (рекомендуется для сложных агентов)
Используй Claude Agent SDK (@anthropic-ai/claude-agent-sdk) когда агенту нужно:
- Выполнять код, работать с файлами
- Использовать MCP-серверы для интеграций
- Выполнять сложные многошаговые задачи
- Координировать субагентов

Архитектура:
```
Сервер (Next.js API Route / Node.js на VPS)
  └── Claude Agent SDK
        ├── Системный промпт (роль, ограничения)
        ├── Инструменты (MCP, custom tools)
        ├── Контекст пользователя (из БД)
        └── Результат → ответ пользователю
```

Установка: npm install @anthropic-ai/claude-agent-sdk
Аутентификация: ANTHROPIC_API_KEY (НЕ claude.ai логин)

### Подход B: Anthropic Messages API (рекомендуется для простых агентов)
Используй прямой API (@anthropic-ai/sdk) когда агенту нужно:
- Отвечать на вопросы пользователей
- Классифицировать, анализировать, генерировать текст
- Вызывать инструменты (Tool Use)
- Работать в режиме чата

Архитектура:
```
Сервер (API Route / Node.js на VPS)
  └── Anthropic SDK → messages.create()
        ├── system: "Ты — агент поддержки компании X..."
        ├── tools: [{ name: "search_docs", ... }]
        ├── messages: [история чата]
        └── Результат → стрим в UI
```

Установка: npm install @anthropic-ai/sdk

## Структура файлов для продакшн агентов
```
src/
  agents/
    config/           — конфигурации агентов (промпты, tools)
    tools/            — кастомные инструменты для агентов
    handlers/         — обработчики запросов к агентам
    types.ts          — типы для агентов
  app/
    api/agents/       — API-роуты для взаимодействия с агентами
```

## Экономика продакшн агентов
- Стоимость: $3/$15 за 1M токенов (Sonnet) или $5/$25 (Opus)
- Кэширование промптов экономит до 90% на повторных вызовах
- Используй Sonnet для большинства задач, Opus только для сложных
- Суммируй стоимость в расчёте на одного пользователя в месяц
- Закладывай токены в pricing модели SaaS

## Чеклист
- [ ] API-ключ в переменных окружения, не в коде
- [ ] Rate limiting для пользовательских запросов к агенту
- [ ] Логирование всех взаимодействий (для отладки и аналитики)
- [ ] Стриминг ответов для UX
- [ ] Graceful degradation если API недоступен
- [ ] Оценка стоимости на пользователя
```
